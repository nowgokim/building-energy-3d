# EnergyPlus 시뮬레이션 통합 계획

> **작성일**: 2026-03-28
> **참조**: `8.simulation/Korean_BB/` (146K 시뮬 결과), `8.simulation/blender/` (BuildWise EUI 룩업)
> **목표**: 현재 `ARCHETYPE_PARAMS` 수작업 EUI → Korean_BB EnergyPlus 통계 기반 EUI로 교체

---

## 1. 현황 분석

### 1.1 현재 EUI 산출 방식

`src/simulation/archetypes.py`의 `ARCHETYPE_PARAMS`에 83개 (9용도 × 4연대 × 2~3구조) 수작업 추정치 하드코딩.

문제점:
- 근거 불투명 (문헌 기반 추정, 불확도 미정량화)
- 서울 기후 고정 (지역 보정 없음)
- 구조 유형별 차이 과장 가능성

### 1.2 Korean_BB 시뮬레이션 자산

| 항목 | 내용 |
|------|------|
| 총 시뮬 디렉토리 | **146,110건** (`simulations/npy_tier_a/`) |
| 아키타입 | 14종: `apartment_highrise`, `apartment_midrise`, `hospital`, `hotel`, `large_office`, `office`, `restaurant_full`, `restaurant_quick`, `retail`, `school`, `small_office`, `strip_mall`, `university`, `warehouse` |
| 연대(vintage) | 5단계: `v1_pre1990`, `v2_1991_2000`, `v3_2001_2010`, `v4_2011_2017`, `v5_2018_plus` |
| 도시 | 5개: `busan`, `daegu`, `gangneung`, `jeju`, `seoul` |
| 고유 조합 | **350개** (14 × 5 × 5 — 일부 결측) |
| 파라메트릭 샘플 | 조합당 평균 418건 (12D LHS) |
| 출력 변수 | `hourly_heating`, `hourly_cooling`, `hourly_electricity`, `hourly_gas` 등 17개 `.npy` |

### 1.3 아키타입 매핑 (building-energy-3d → Korean_BB)

| building-energy-3d `usage_type` | Korean_BB 아키타입 | 조건 |

| `office` (≥15K m² or ≥10층) | `large_office` | |
| `office` (≥3K m²) | `office` | |
| `office` (<3K m²) | `small_office` | |
| `retail` (단독) | `retail` | |
| `retail` (복합·상가) | `strip_mall` | |
| `education` (초·중·고) | `school` | |
| `education` (대학) | `university` | |
| `hospital` | `hospital` | |
| `warehouse` | `warehouse` | |
| `hotel` | `hotel` | |
| `cultural`, `mixed_use`, `미분류` | `office` (기본값) | |

### 1.4 연대 매핑

| building-energy-3d | Korean_BB |
|-------------------|-----------|
| `pre-1980` | `v1_pre1990` |
| `1980-2000` | `v2_1991_2000` |
| `2001-2010` | `v3_2001_2010` |
| `post-2010` | `v4_2011_2017` (2011~2017) 또는 `v5_2018_plus` |

`post-2010` → `built_year >= 2018` 이면 `v5`, 그 외 `v4` 로 세분화.

## 2. 구현 계획

### Phase 0 — Korean_BB NPY에서 EUI 통계 추출 (즉시 착수 가능)

**목표**: 146K 시뮬에서 EUI(kWh/m²/yr) 통계(중앙값, P10, P90) 추출 → `src/simulation/eui_table.py` 자동 생성

**신규 파일**: `src/data_ingestion/extract_eui_from_korean_bb.py`

```python
"""
Korean_BB npy_tier_a에서 EUI 통계를 추출하여
src/simulation/eui_table.py를 자동 생성한다.

실행:
    python -m src.data_ingestion.extract_eui_from_korean_bb \
        --npy-dir "C:/Users/User/Desktop/myjob/8.simulation/Korean_BB/simulations/npy_tier_a" \
        --out src/simulation/eui_table.py
"""
```

추출 로직:
1. 각 시뮬 디렉토리에서 `metadata.json` (archetype, vintage, city) 읽기
2. `hourly_heating.npy` + `hourly_cooling.npy` + `hourly_electricity.npy` + `hourly_gas.npy` 합산 → annual EUI (kWh/m²/yr)
3. `total_area`는 `metadata.json`의 `floor_area` 필드 사용 (없으면 1로 정규화)
4. 조합별 `(median, p10, p90, n_samples)` 집계
5. 서울(`seoul`) 기후 기준 중앙값을 primary EUI로 사용

출력 형식 (`eui_table.py`):
```python
# AUTO-GENERATED by extract_eui_from_korean_bb.py — DO NOT EDIT
# Source: Korean_BB npy_tier_a (146K EnergyPlus simulations)
# Generated: 2026-03-28

EUI_TABLE: dict[tuple[str, str, str], dict] = {
    ("apartment_highrise", "v1_pre1990", "seoul"): {
        "eui_median": 142.3,  # kWh/m²/yr
        "eui_p10": 108.5,
        "eui_p90": 189.2,
        "n": 418,
    },
    ...
}
```

### Phase 1 — archetypes.py EUI 교체

**목표**: `ARCHETYPE_PARAMS`의 `ref_total`을 Korean_BB 통계로 교체

**수정 파일**: `src/simulation/archetypes.py`

```python
from src.simulation.eui_table import EUI_TABLE

def get_eui_from_korean_bb(
    usage_type: str,
    vintage_class: str,
    floors_above: int,
    total_area: float,
    city: str = "seoul",
) -> dict[str, float]:
    """Korean_BB EUI 통계를 반환. 매핑 실패 시 ARCHETYPE_PARAMS fallback."""
    korean_bb_archetype = _map_to_korean_bb_archetype(usage_type, floors_above, total_area)
    korean_bb_vintage = _map_vintage(vintage_class)
    key = (korean_bb_archetype, korean_bb_vintage, city)
    if key in EUI_TABLE:
        return EUI_TABLE[key]
    # 도시 fallback: 서울
    key_seoul = (korean_bb_archetype, korean_bb_vintage, "seoul")
    if key_seoul in EUI_TABLE:
        return EUI_TABLE[key_seoul]
    return None  # 상위 호출자에서 ARCHETYPE_PARAMS로 fallback
```

### Phase 4 (선택) — 갭 커버 신규 시뮬레이션

Korean_BB에 없는 아키타입:
- `residential_single` → 단독주택 IDF 신규 제작 필요
- `cultural`, `mixed_use` → 복합용도 IDF

신규 IDF 제작은 `src/simulation/idf_generator.py`에서 Korean_BB `IDFModifier` 래퍼 활용.

## 4. 우선순위 및 의존성

```
Phase 0 (extract_eui_from_korean_bb.py)
    ↓
eui_table.py 생성
    ↓
Phase 1 (archetypes.py 수정)
    ↓
Phase 2 (DB 마이그레이션)
    ↓
Phase 3 (배치 업데이트) → REFRESH MV
```

Phase 0은 외부 의존성 없이 즉시 착수 가능.
Phase 2~3은 Docker Compose DB 실행 중일 때만 가능.

## 6. 미결 사항

- `total_area`가 없는 시뮬 디렉토리의 EUI 정규화 방법 → Korean_BB `metadata.json` 필드 확인 필요

- 서울 이외 지역 건물(경기도 등)의 도시 기후 매핑 테이블 작성 필요
