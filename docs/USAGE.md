## 3. 레이어 버튼 상세 설명

### 3.1 🎨 면 텍스처

VWorld 정부 공식 LoD3-4 텍스처 건물을 켜고 끕니다.

- **ON**: 실제 건물 외관 사진 텍스처 렌더링 (서울 주요 지역)
- **OFF**: 기본 3D 건물 형태만 표시
- 텍스처 건물이 없는 지역에서는 차이가 없습니다.

### 3.2 🔥 화재위험도

건물별 화재 위험 점수를 색상으로 오버레이합니다.

- **고위험(빨강, ≥60점)**: 목조·조적조, 1980년 이전 건축, 위험물·창고 용도
- **중위험(주황, 35~59점)**: RC 구조 또는 2000년대 이전 건물
- **저위험(초록, <35점)**: 철근콘크리트, 최근 건축, 저층 주거

> **주의**: 카메라 고도가 5km 초과 시 "더 가까이 확대하면 표시합니다" 메시지가 나타납니다.
> 화면을 확대하면 자동으로 레이어가 로드됩니다.

**점수 산정 기준**:

| 항목 | 배점 | 고위험 조건 |
|------|------|-----------|
| 구조 유형 | 40점 | 목조·조적조 > 경량철골 > RC·철골 |
| 건축연도 | 30점 | 1970년 이전 > 1980년대 > 1990년대 |
| 용도 | 20점 | 위험물·창고 > 노유자·의료 > 판매·주거 |
| 층수 | 10점 | 15층 초과 > 5~15층 > 5층 이하 |

### 3.3 🔴 화재 취약 밀집구역

화재 고위험 건물(점수 ≥60)이 반경 100m 이내에 5동 이상 밀집된 구역을 폴리곤으로 표시합니다.

- **빨강 반투명 폴리곤**: 고위험 클러스터 (평균 점수 ≥60)
- **주황 반투명 폴리곤**: 중위험 클러스터 (평균 점수 35~60)
- 폴리곤은 지형 위 120m 높이의 3D 볼륨으로 표시됩니다.
- 서울 전역 기준 **991개 클러스터** (2026-03-25 계산 기준)

> 마포, 성북, 도봉, 노원 지역에 고밀 클러스터가 집중됩니다.

### 3.4 🚒 소방서 위치

서울시 소방서 본서 25개의 위치를 빨간 원 마커로 표시합니다.

- 마커 클릭 시 소방서 이름, 자치구, 유형을 표시합니다.
- 향후 119안전센터 119개를 추가할 예정입니다.

## 4. 건물 클릭 — 상세 패널

지도에서 건물을 클릭하면 우측 하단에 상세 정보 패널이 열립니다.

```
┌────────────────────────────────┐
│ 마포구 서교동 367-1             │
│ PNU: 1144010100...             │
│                                │
│ [에너지] EUI: 185 kWh/m²·yr    │
│ [화재]   위험도: 75점 (고위험) │
│                                │
│ 🚒 최근접 소방서:               │
│    마포소방서 (2.5분 / 1.2km) │
└────────────────────────────────┘
```

**소방서 응답시간 색상**:
- 초록: 5분 이내
- 주황: 5~10분
- 빨강: 10분 초과

## 5. API 엔드포인트 레퍼런스

### 5.1 화재 위험도

```
GET /api/v1/fire/risk?west=126.9&south=37.5&east=127.0&north=37.6
GET /api/v1/fire/risk/{pnu}
GET /api/v1/fire/stats?west=...&south=...&east=...&north=...
```

### 5.2 소방서

```
GET /api/v1/fire/stations
GET /api/v1/fire/coverage/{pnu}    # 최근접 소방서 3개 + 응답시간
GET /api/v1/fire/adjacency/{pnu}   # 25m 이내 인접 건물 목록
```

### 5.3 클러스터

```
GET /api/v1/fire/clusters?risk_level=HIGH
POST /api/v1/fire/clusters/compute?eps_m=100&min_points=5&min_score=60
```

### 5.4 인접 그래프 빌드 (Celery 태스크)

```
POST /api/v1/fire/adjacency/build?sgg_code=11440&dist_m=25
POST /api/v1/fire/adjacency/build    # 서울 전체 25개 구 순차 실행
```

### 5.5 화재 확산 시뮬레이션 (Phase F2)

```
POST /api/v1/fire/scenario           # 시나리오 실행 (origin_pnu, wind_direction, wind_speed)
GET  /api/v1/fire/scenario/{id}      # 결과 조회 (타임라인 JSON)
GET  /api/v1/fire/scenarios          # 저장된 시나리오 목록
```

요청 예시:
```json
{ "origin_pnu": "1144010100...", "wind_direction": 270.0, "wind_speed": 5.0, "max_steps": 20 }
```

### 5.6 대피 경로 (Phase F3)

```
GET /api/v1/fire/evacuation/{pnu}?max_routes=3
```

반환: 가장 가까운 집결지 3개까지의 Dijkstra 최단 경로 (LineString 좌표 배열, 거리(m)).
집결지: 서울 25개 구 구민체육관 + 대형공원·광장 (총 35개).

### 5.7 기상 (Phase F4)

```
GET  /api/v1/fire/weather/current      # 현재 서울 풍향·풍속 (JSON)
POST /api/v1/fire/weather/collect      # 즉시 수집 트리거 (Celery beat 외 수동)
WS   /ws/weather                       # WebSocket: 60초 간격 push
```

화재 확산 시뮬레이션 실행 시 기상 WebSocket이 wind_direction/wind_speed를 자동 세팅합니다.

Swagger UI: `http://localhost:8000/docs`
