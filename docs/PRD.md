# PRD: 3D 건물 에너지 시뮬레이션 지도 플랫폼

**문서 버전**: 1.1

**최종 수정**: 2026-03-21 (5개 분야 전문가 리뷰 반영)
**상태**: Draft

## 1. 개요 & 배경

### 1.1 문제 정의

한국의 건물 에너지 관리는 다음과 같은 문제를 가진다:

- **시각화 부재**: 건물 에너지 데이터가 2D 표/차트로만 제공되어 직관적 파악이 어려움
- **데이터 분절**: 건축물대장, 에너지소비량, GIS 공간정보가 각각 별도 시스템에 산재
- **시뮬레이션 접근성 부족**: EnergyPlus 등 시뮬레이션 도구는 전문가만 사용 가능
- **도시 스케일 분석 한계**: 개별 건물 단위 분석은 가능하나 도시/지구 단위 에너지 패턴 파악 도구 부재
- **정책 도구 부족**: 제로에너지건축물, 에너지효율등급 등 정책 시행을 위한 3D 시각적 의사결정 도구 없음

### 1.2 시장 맥락

- 글로벌 디지털 트윈 시장: $18.9B(2025) → $428.1B(2034), CAGR 41.4%
- 빌딩 트윈 시장: $4.18B(2026)
- 한국 공간정보 정책사업 예산: 5,838억원(2025), 70%가 디지털트윈 구축
- 국토부 디지털트윈국토 16개 국가표준 수립 완료
- 기존 엔터프라이즈 제품(Autodesk Tandem $3,425/년, Bentley iTwin)은 고가이며 한국 데이터 연동 미지원

### 1.3 솔루션 개요

한국 정부 건축물 데이터(건축물대장, GIS건물통합정보)를 활용하여 건물을 3D로 시각화하고, 각 건물의 외피/창호 정보 기반 에너지 시뮬레이션 결과를 지도 위에 직관적으로 표시하는 웹 플랫폼.

## 2. 목표 & 성공 지표

### 2.1 프로젝트 목표

1. 한국 정부 건축물 데이터를 자동 수집하여 3D 건물 모델을 생성하는 파이프라인 구축
2. 건물 외피/창호 속성 기반 에너지 시뮬레이션 실행 및 결과 시각화
3. 도시/지구 스케일의 건물 에너지 패턴을 3D 지도에서 직관적으로 탐색
4. 에너지 컨설턴트, 도시 계획가, 건물 관리자가 즉시 활용할 수 있는 웹 인터페이스

### 2.2 성공 지표 (KR)

| 지표 | 목표 |
|------|------|
| 대상 지역 건물 3D 렌더링 커버리지 | 95% 이상 |
| 3D 지도 초기 로딩 시간 | 5초 이내 |
| 건물 클릭 → 에너지 정보 표시 | 2초 이내 |
| 지원 건물 수 (MVP 대상 지역) | 10,000+ |
| 에너지 시뮬레이션 vs 실제 소비 오차 | 30% 이내 (원형 기반) |

### 2.3 비목표 (Scope 제외)

- 실시간 IoT 센서 연동 (Phase 2)
- 실내 공간 3D 모델링 (LoD4)
- 건물 구조 안전 분석
- BIM(IFC) 파일 직접 업로드 편집 (Phase 2)
- 모바일 네이티브 앱 (웹 반응형으로 대체)

## 3. 타겟 사용자 & 페르소나

### 3.1 도시 계획가 (Primary)

**"박 주무관" - 시청 도시계획과 공무원**

- 도시 재개발, 에너지 정책 수립, 제로에너지건축물 정책 시행
- **Pain**: 지역 내 노후 건물 에너지 실태를 한눈에 파악 불가. 정책 보고서용 시각 자료 제작에 수주 소요
- **Need**: 건축년도별/에너지등급별 건물 분포 3D 시각화, 지역 에너지 통계, 노후 건물 필터링
- **Value**: 데이터 기반 정책 수립, 시민 소통용 시각 자료, 그린리모델링 대상 선정
- **MVP 핵심 기능**: 3D 지도 + 에너지등급 오버레이 + 필터링 + 통계 대시보드

### 3.2 에너지 컨설턴트 (Secondary)

**"김 과장" - 건물 에너지 컨설팅 회사 5년차**

- 건물 에너지 효율등급 인증 업무 수행
- 현재 ECO2 도구와 엑셀로 개별 건물 분석
- **Pain**: 고객에게 지역 전체 건물 에너지 현황을 보여줄 시각적 도구 없음
- **Need**: 3D 지도에서 에너지 등급별 색상 표시, 리트로핏 시나리오 비교
- **Value**: 고객 프레젠테이션 품질 향상, 분석 시간 단축
- **MVP 포함 기능**: 간소화된 리트로핏 추정 (창호 교체/외단열 추가 시 등급 변화 예측)

### 3.3 건물 관리자

**"이 팀장" - 대형 상업 건물 시설관리 담당**

- 건물 에너지 비용 절감 KPI
- **Pain**: 건물 외피 어디서 에너지 손실이 큰지 직관적으로 모름
- **Need**: 자기 건물의 벽체/창호별 열관류율, 에너지 소비 패턴 시각화
- **Value**: 에너지 절감 투자 우선순위 결정

### 4.1 한국 정부 데이터 인벤토리

| 데이터 | API/소스 | 제공 정보 | 포맷 | 갱신주기 | 접근 방식 |

### 4.2 핵심 데이터 제약사항

1. **건축물대장에 지오메트리 없음**: 속성만 제공. 건물 폴리곤은 GIS건물통합정보에서 취득
2. **VWorld 3D 메시 벌크 다운로드 불가**: 2019년 폐쇄 (국가보안). 자체 3D 모델 생성 필수
3. **Google Photorealistic 3D Tiles 한국 미지원**: 2026-03 확인. Cesium OSM Buildings (Ion 96188)로 대체
4. **건축물대장 총괄표제부만으로는 동별 정보 부족**: 표제부(getBrTitleInfo) 추가 수집 필요 (구조, 준공일, 동별 층수)
3. **API Rate Limit**: 개발계정 1,000건/일, 운영계정 100,000건/일 → 초기 벌크 SHP 다운로드 후 API는 델타 동기화용
4. **데이터 품질 편차**: 지자체별 필드 누락/형식 불일치. 특히 구축 시기가 다른 읍/면/동
5. **조인 키**: PNU 코드 (19자리: 시도2+시군구3+읍면동3+리2+산대지1+본번4+부번4)
6. **집합건물(아파트) 1:N 매칭**: 1개 PNU에 수십~수백 개 건축물대장 레코드 존재. 총괄표제부(단지)+표제부(동별)+전유부(세대별) 계층 구조. **PNU + 건물관리번호(25자리) 복합키** 필요
7. **도로명주소↔PNU 매핑**: 사용자는 도로명주소로 검색 → 행안부 도로명주소 API로 PNU 변환 필수

### 4.3 에너지 시뮬레이션 입력 데이터 매핑

건축물대장이 제공하지 않는 에너지 파라미터를 추정하는 매핑 (핵심 IP):

| 파라미터 | 추정 방법 |
|----------|----------|
| 벽체/지붕 U-value | 건축년도 → 국토부 고시 「건축물의에너지절약설계기준」 이력별 기준값 × **열화계수**(10년당 1.3배 증가) 자동 매핑 (중부1/중부2/남부/제주) |
| 창호 U-value | 건축년도 + 지역 → 고시 기준 × 열화계수 (아르곤 누출, Low-E 열화, 실란트 손상) |
| 창면적비(WWR) | 건물 용도별 기본값 (주거 20~30%, 사무 30~50%, 상업 50~70%) |
| HVAC 시스템 유형 | 건물 용도 + 건축년도 + 규모별 추정. **한국 공동주택: 온돌(바닥복사난방)** 모델링 필수 |
| 내부 부하 프로파일 | 용도별 ASHRAE 90.1 / 한국 기준 기본값 |
| 기후 데이터 | KIAEBS 표준기상데이터(S-19:2024) + **도시열섬(UHI) 보정** (서울 가을 최대 4.3°C) |
| 발코니 확장 여부 | 2005년 이후 아파트는 발코니 확장 가정 (열성능 7~15% 영향) |

### 4.4 건물 원형(Archetype) 분류 체계

도시 스케일 시뮬레이션을 위해 건물을 200~500개 원형으로 분류:

- **축 1**: 용도 (주거-아파트/단독, 사무, 상업, 교육, 의료 등) ~10종
- **축 2**: 건축년도 구간 (~2001, 2001~2010, 2010~2017, 2017~) ~4구간
- **축 3**: 규모 (소/중/대) ~3등급
- **축 4**: 구조 유형 (RC/철골/조적/목구조) ~3종 (RC+철골 통합 가능. **구조간 열성능 차이 30%+**)
- **축 5**: 기후 지역 (중부1/중부2/남부/제주) ~4지역
- 조합: 10 x 4 x 3 x 3 x 4 = 1,440 (실 존재 조합은 200~500개)

### 6.1 성능

| 항목 | 목표 |

### 6.2 3D 타일 최적화 전략 (HLOD)

도시 스케일 성능을 위한 3단계 계층적 LOD:

| 단계 | geometricError | 내용 | 타일당 건물 수 |
|------|---------------|------|---------------|
| Tier 1 (도시 개요) | > 100m | 블록 단위 병합 메시, vertex color | 전체 도시 수십 개 타일 |
| Tier 2 (지구 레벨) | 10~100m | LoD1 건물 배치, 텍스처 아틀라스 | 50~200건물/타일 |
| Tier 3 (거리 레벨) | < 10m | 개별 LoD2/LoD3 건물, PBR 재질 | 1~10건물/타일 |

최적화: 양자화 정점(16-bit), oct-encoded 법선, Draco/meshopt 압축, gzip 전송

### 6.3 데이터 동기화

### 7.1 핵심 플로우: 지역 에너지 현황 탐색

```
1. 웹 접속 → 마포구 2D 지도 + 검색바 표시 (3D 글로브 대신 업무 효율 우선)
2. 도로명주소 검색 → 해당 위치로 이동 + 3D 전환
3. 에너지등급 레이어 활성화 → 건물별 색상 표시
   (녹색: 1+++등급 ~ 빨강: 7등급)
4. 건물 클릭 → 우측 패널에 상세 정보
   - 기본 정보 (주소, 용도, 면적, 건축년도)

   - 외피 정보 (벽체/창호 U-value)
   - 에너지 분해 차트 (냉방/난방/급탕/조명/환기)
5. 필터 적용 → "2000년 이전 + 5등급 이하" 건물만 표시
6. 통계 패널 → 현재 뷰의 평균 에너지 소비, 등급 분포
7. CSV 내보내기 → 필터된 건물 목록 다운로드
```

## 8. 시스템 아키텍처 (요약)

### 8.1 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| **프론트엔드** | React + CesiumJS (Resium) | 3D 글로브 + 3D Tiles 네이티브 |
| **백엔드** | Python (FastAPI) 모듈러 모놀리스 | 한국 API 연동, 에너지 엔진 연동, 추후 분리 용이 |
| **3D 생성** | trimesh (LoD1/2 벌크) | Blender 대비 100배 가볍고 빠름, GPU 불필요 |
| **3D Tiles 변환** | pg2b3dm / py3dtilers | PostGIS → 3D Tiles 직접 변환 |
| **에너지 엔진** | OpenStudio SDK + EnergyPlus | 원형별 시뮬레이션, URBANopt 연동 |
| **ML 대리모델** | XGBoost/LightGBM (사용자 추후 대체) | 도시 스케일 빠른 추론용 플레이스홀더 |
| **DB** | PostgreSQL + PostGIS | 공간 데이터 + 건물 속성 통합 |
| **베이스맵** | PMTiles (Planetiler 생성) | 타일 서버 인프라 불필요, S3 정적 호스팅 |
| **3D Tiles 서빙** | S3 + CDN (CloudFront) | 정적 파일 서빙, 서버 부하 없음 |
| **작업 큐** | Celery + Redis | 데이터 동기화, 시뮬레이션 비동기 실행 |
| **내부 3D 포맷** | CityJSON | CityGML 대비 7배 컴팩트, JSON 네이티브 |

### 8.2 데이터 파이프라인

```

  2D 폴리곤 footprint           속성 데이터
        │                          │
        ▼                          ▼
  ogr2ogr → PostGIS           Python → PostgreSQL
        │                          │
        └───── PNU 코드 JOIN ──────┘
                    │
                    ▼
           통합 Materialized View
           (geom + 높이 + 용도 + 에너지등급)
                    │
            ┌───────┴───────┐
            ▼               ▼
    ST_Extrude (3D)    에너지 원형 매칭
            │               │
            ▼               ▼
    pg2b3dm → 3D Tiles  OpenStudio 시뮬레이션
            │               │
            ▼               ▼
    S3 + CDN 배포      결과 → PostGIS 저장
            │               │
            └───────┬───────┘
                    ▼

            (3D 건물 + 에너지 색상)
```
